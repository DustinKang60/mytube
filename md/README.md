## 핵심 요약

**가장 중요한 결론 3가지:**

1. **끊김의 원인은 터널이 아니라 googlevideo의 연결당 실시간 제한(pacing)이었다.**
   → 터널을 ngrok 등으로 바꿔도 소용없다. **병렬 다운로드로 우회**하는 게 답이었다.

2. **클라이언트 재연결로는 화면 끄기를 이길 수 없다.**
   브라우저가 백그라운드에서 타이머·네트워크를 얼려버린다.
   3번 시도했다 전부 실패했고, 관련 코드는 제거했다. **다시 넣지 말 것.**

3. **서비스워커가 캐시 우선이라 새 코드가 바로 안 보인다.**
   에셋 바꾸면 `sw.js`의 `CACHE_NAME` 버전을 반드시 올리고,
   폰에선 앱 완전 종료 후 새로고침해야 반영된다. 이걸 모르면 "수정 실패"로 오판한다.

## 현재 구조 한 줄 요약

```
목록: 유튜브 RSS → 서버 /fetch(또는 공개 프록시) → 폰 localStorage에 6시간 캐시
재생: 서버가 16개 병렬 연결로 통째 다운로드 → 디스크 캐시 → 폰이 통째로 받아 메모리(Blob)에서 재생
      → 네트워크 불필요 → 화면 꺼도 재생 유지
폴백: 서버 없거나 실패 시 유튜브 임베드(광고 있음, 화면 켜야 함)
```
