# mytube

유튜브 채널의 오디오만 골라 듣는 개인용 라디오 PWA. **작은앱공방**

구독한 채널의 최근 영상을 목록으로 받아, **화면을 꺼도 오디오가 계속 재생**됩니다.
강연·라디오·팟캐스트처럼 "듣기만 하면 되는" 유튜브 콘텐츠용입니다.

**앱 열기**: https://dustinkang60.github.io/mytube/
(설치 불필요 — 브라우저에서 열고 "홈 화면에 추가" 하면 앱처럼 쓸 수 있습니다.)

---

## 뭘 할 수 있나

- 유튜브 채널 구독 (채널 ID / URL / `@핸들` / 이름으로 추가)
- 채널의 최근 영상 목록 → 탭해서 오디오 재생
- **화면을 꺼도, 다른 앱을 써도 재생 유지** (개인 서버 사용 시)
- 잠금화면 / 알림에서 재생·일시정지·이전·다음 조작
- 재생 속도 조절 (1x / 1.25x / 1.5x / 2x) — 강연 들을 때 유용
- **이어듣기 / 한 곡만** 모드 — 음악은 계속, 강연은 하나 듣고 끝
- "더 보기"로 오래된 영상까지 탐색

## 두 가지 재생 방식

| | 개인 서버 있음 | 서버 없음 (기본) |
|---|---|---|
| 광고 | **없음** | 있음 |
| 화면 끄고 재생 | **가능** | 불가 (멈춤) |
| 준비 | 서버 폰 설정 필요 | 없음 — 바로 사용 |

**서버 없이도 바로 쓸 수 있지만**, 화면을 끄면 멈추고 광고가 나옵니다(유튜브 임베드 정책).
안 쓰는 안드로이드폰 하나를 서버로 두면 이 제약이 모두 사라집니다.

👉 **서버 설정 방법: [`server/README.md`](server/README.md)**

## 어떻게 동작하나

```
목록  유튜브 RSS 피드 → (서버 /fetch 또는 공개 프록시) → 폰에 6시간 캐시
재생  서버가 16개 병렬 연결로 오디오를 통째 다운로드 → 디스크 캐시
      → 앱이 완성된 파일을 통째로 받아 메모리(Blob)에서 재생
      → 재생 중 네트워크 불필요 → 화면 꺼도 유지
폴백  서버가 없거나 실패하면 유튜브 임베드 (광고 있음, 화면 켜야 함)
```

유튜브는 연결 하나당 **실시간 배속(~30KB/s)** 으로만 오디오를 보내서 스트리밍으로는
버퍼를 앞서 쌓을 수 없고, 화면을 끄면 브라우저가 네트워크를 멈춰 재생이 끊깁니다.
이 제한이 **연결마다** 걸린다는 점을 이용해 병렬로 나눠 받아 우회했습니다
(**5.5MB/s** — 1시간짜리 64.9MB가 12초).

자세한 진단 과정과 설계 근거: [`md/`](md/)

## 구성

| 파일 | 역할 |
|---|---|
| `index.html`, `style.css`, `app.js` | 앱 본체 (프레임워크 없는 정적 PWA) |
| `sw.js` | 서비스워커 (오프라인 캐시). **에셋 수정 시 `CACHE_NAME` 버전을 올릴 것** |
| `manifest.json`, `icons/` | PWA 설정 |
| `server/mytube-server.py` | 개인 서버 (오디오 병렬 다운로드 + 캐시 + 서빙) |
| `server/setup.sh` | Termux 설치 스크립트 |
| `server/boot-setup.sh` | 재부팅 자동시작 설정 |
| `md/` | 개발 기록 — 왜 이렇게 만들었는지, 뭘 시도했다 실패했는지 |

## 개발 시 주의

- **서비스워커가 캐시 우선**입니다. `app.js` 등을 고치면 `sw.js` 의 `CACHE_NAME` 을
  반드시 올리세요. 안 그러면 기기가 옛 코드를 계속 씁니다 (수정이 실패한 것처럼 보임).
- 폰에서 확인할 땐 앱을 **완전히 종료**(최근 앱에서 스와이프)한 뒤 다시 열고 새로고침하세요.
- 서버는 **가정용 IP**에서 돌려야 합니다. 데이터센터/VPS 는 유튜브 봇 차단에 걸립니다.
- **라이브 방송은 지원하지 않습니다** (yt-dlp 가 m4a 를 뽑지 못함).
